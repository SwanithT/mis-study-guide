Chapter 9: Cloud Computing, SaaS, and AWS

Key concepts, definitions, and scenario-based questions.

---

Vocabulary

Cloud Computing: Delivering computing resources (servers, storage, software) over the internet instead of owning them locally.
Example: Using Google Docs instead of Microsoft Word installed on your laptop.
Software as a Service (SaaS): Software hosted on remote servers and accessed through the internet.
Example: Canvas, Google Docs, or Office 365.
Hosted Software: Software that runs on remote servers and requires internet access.
Example: Google Sheets running in your browser.
Local Software: Software that runs directly on your device without needing a network.
Example: Excel downloaded on your laptop.
Infrastructure as a Service (IaaS): Cloud service where companies rent hardware and manage their own software.
Example: Netflix running on AWS servers.
Platform as a Service (PaaS): Cloud service where companies rent infrastructure and platform tools but build their own applications.
Example: Salesforce platform for building apps.
Amazon Web Services (AWS): A major cloud provider offering computing infrastructure and services.
Example: Many companies host their apps on AWS.
Service Level Agreement (SLA): A contract defining system performance, uptime, and reliability.
Example: 99.9% uptime guarantee.
Total Cost of Ownership (TCO): The total cost of a system over time, including purchase, maintenance, and support.
Example: SaaS lowers upfront cost but includes ongoing subscription fees.
Latency: Delay between a user action and system response due to network communication.
Example: Slow loading when using cloud apps on weak WiFi.
Scalability: Ability to increase or decrease system resources based on demand.
Example: A website handling traffic spikes automatically.
Freemium Model: Offering basic services for free and charging for premium features.
Example: Spotify free vs premium.
Vendor Dependence: Reliance on a single provider for critical systems.
Example: A company relying entirely on AWS.
Switching Costs: The cost and difficulty of moving from one system/provider to another.
Example: Migrating data from one cloud provider to another.
