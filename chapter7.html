Chapter 7: Understanding Software

Key concepts, definitions, and scenario-based questions.

---

Vocabulary

Application: Software designed to perform specific tasks for users.
Example: Excel is used for data analysis and calculations.
Operating System: Software that manages hardware and provides services for applications.
Example: Windows and macOS allow apps to run and manage files.
Database Management System (DBMS): Software used to store, organize, and retrieve structured data.
Example: A university system storing student records.
Middleware: Software that connects different systems or applications so they can communicate.
Example: Middleware allows a website to access a database.
Enterprise Resource Planning (ERP): Systems that integrate core business processes into one unified system.
Example: Tracking inventory, accounting, and HR in one platform.
Customer Relationship Management (CRM): Systems used to manage interactions with customers and track relationships.
Example: Salesforce tracking customer sales and interactions.
Business Intelligence (BI): Tools and systems used to analyze data and support decision-making.
Example: Dashboards showing company performance metrics.
Supply Chain Management (SCM): Systems that manage the flow of goods, information, and finances across suppliers and customers.
Example: Tracking shipments from suppliers to stores.
User Interface (UI / GUI): The part of software that users interact with.
Example: Buttons and menus on a mobile app.
Distributed Computing: Processing that occurs across multiple machines instead of one central system.
Example: Cloud systems using many servers to process tasks.
Compatibility: The ability of software to run on different systems or hardware.
Example: An app working on both Windows and Mac.
Integration: The process of combining different systems so they work together.
Example: Linking CRM with email systems.
