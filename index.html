Chapter 6: Moore’s Law & Hardware

Vocabulary

Moore’s Law: The observation that computing power increases exponentially over time, meaning chips become faster and cheaper at a predictable rate.
Price Elasticity (Technology): The idea that as computing power becomes cheaper, demand increases significantly because people find new uses for it.
Konana’s Software Ecosystem: A layered system where hardware, operating systems, and applications depend on each other, creating switching costs and user lock-in.
CPU (Central Processing Unit): The main processor that executes instructions and performs calculations in a computer.
Memory (RAM): Temporary storage used by the CPU to quickly access data currently in use.
Storage: Long-term data storage that keeps information even when the computer is turned off.
Latency: The time delay before data begins transferring between systems.
Bandwidth: The amount of data that can be transmitted over a network in a given time.
e-Waste: Discarded electronic devices that can harm the environment due to toxic materials.
---

Key Concepts

Computing power continues to grow rapidly due to Moore’s Law, even as physical limits are reached. As computing becomes cheaper, demand increases because of price elasticity. Modern computing relies heavily on cloud processing, which introduces latency but allows for large-scale parallel computing. Systems are interconnected through software ecosystems, making switching costly for users.

Practice Questions

1. A student startup creates an app that requires heavy AI processing. Instead of running everything locally, they send data to cloud servers. Users complain that responses take a few seconds to load.

Which concept BEST explains this delay?

A. Bandwidth limitations B. Latency C. Storage capacity D. Price elasticity

Correct answer: B

Latency refers to the delay in data traveling to and from the cloud, which causes slower response times. Bandwidth relates to how much data can be transferred, not the delay itself. Storage capacity is unrelated to speed of response. Price elasticity explains demand changes, not performance issues.

---
2. A company notices that as cloud storage becomes cheaper, customers begin storing significantly more data and using more services.

Which concept BEST explains this behavior?

A. Switching costs B. Moore’s Law C. Price elasticity D. Vertical integration

Correct answer: C

Price elasticity explains how demand increases when prices decrease. Moore’s Law explains technological growth, not user behavior. Switching costs relate to difficulty changing systems. Vertical integration refers to owning multiple stages of production.

---
3. A company builds software that only works with its own operating system, making it difficult for users to switch to competitors.

Which concept BEST explains this situation?

A. Network effects B. Switching costs C. Latency D. Multicore processing

Correct answer: B

Switching costs arise when it is difficult for users to move to another system due to dependencies. Network effects relate to value increasing with users. Latency is about delay in data transfer. Multicore processing relates to hardware performance.
