Chapter 8: Open Source Software

Key concepts, definitions, and scenario-based questions.

---

Vocabulary

Open Source Software (OSS): Software where the source code is freely available for anyone to view, modify, and redistribute.
Example: Linux can be modified by developers worldwide.
Closed Source Software: Software where only the owning company can access and modify the source code.
Example: Windows is controlled entirely by Microsoft.
Source Code: The original code written by programmers that defines how software works.
Example: Developers edit source code to fix bugs or add features.
Linux: An open-source operating system that powers much of the internet and cloud infrastructure.
Example: Most servers and even Android devices run on Linux-based systems.
Technology Stack: A set of layered technologies that work together to run an application.
Example: A website may use Linux, a database, and a programming language together.
LAMP Stack: A traditional open-source stack using Linux, Apache, MySQL, and PHP/Python.
Example: Many early websites used LAMP to scale cheaply.
MEAN Stack: A modern stack using MongoDB, Express, Angular, and Node.js.
Example: Apps with real-time updates often use MEAN.
Hosted Software: Software that runs on remote servers and is accessed through the internet.
Example: Google Docs runs in the cloud, not locally.
Local Software: Software that runs directly on a user’s device without needing a network connection.
Example: Microsoft Excel installed on your laptop.
Open Standards: Standards that allow other companies to build compatible products without permission.
Example: Many web technologies allow cross-platform compatibility.
Closed Standards: Standards controlled by a company that require permission for others to build compatible systems.
Example: Apple tightly controls its ecosystem.
Total Cost of Ownership (TCO): The full cost of a system, including maintenance, support, and operation—not just purchase price.
Example: OSS may be free upfront but require support costs later.
Scalability: The ability of a system to handle growth efficiently.
Example: OSS stacks allow companies to scale to millions of users cheaply.
