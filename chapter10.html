Chapter 10: Software Development

Key concepts, definitions, and scenario-based questions.

---

Vocabulary

Systems Development: The process of creating and maintaining information systems, involving people, processes, data, software, and hardware.
Example: Building a company’s internal CRM system.
Requirements: The specific needs and features a system must meet for users.
Example: A system must allow users to log in and track orders.
Scope Creep: The uncontrolled expansion of project requirements during development.
Example: Constantly adding new features delays launch.
Triple Constraint: The trade-off between scope, time, and resources in a project.
Example: Adding features (scope) requires more time or money.
Waterfall Method: A linear development approach where each phase is completed before moving to the next.
Example: Fully planning before any coding begins.
Agile Method: An iterative approach where development happens in small cycles with continuous feedback.
Example: Releasing small updates frequently.
Prototype: A preliminary version of a system used to test ideas and gather feedback.
Example: A mock app interface shown to users.
Wireframe: A simple visual layout of a system interface.
Example: A sketch of a website layout before coding.
Low-Code/No-Code (LCNC): Tools that allow users to build applications with little or no programming.
Example: Microsoft Power Apps.
Citizen Developer: A non-professional programmer who builds applications using LCNC tools.
Example: A business analyst creating an internal tool.
Programming Language: A formal language used to write software instructions.
Example: Python, Java.
Integrated Development Environment (IDE): Software used by programmers to write, test, and debug code.
Example: Visual Studio.
Compilation: Converting code into machine-readable instructions.
Example: C++ code compiled before running.
Interpretation: Running code line-by-line without compiling first.
Example: JavaScript running in a browser.
Compliance: Ensuring systems follow laws and regulations.
Example: Meeting GDPR requirements.
